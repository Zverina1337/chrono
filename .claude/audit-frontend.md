# Аудит Frontend — Chrono

## CRITICAL — ломает функциональность

### 1. `ItemSection.vue` — isDragging без .value
- **Строки:** ~89-90 (в template)
- **Проблема:** `isDragging` это `shallowRef`, но присваивается напрямую `isDragging = true` вместо `isDragging.value = true`
- **Последствие:** Drag-and-drop визуальный фидбек (зелёная рамка, полупрозрачный фон) **не работает**
- **Направление:** Исправить все присваивания в template на `.value`
- [ ] Исправлено

### 2. `ListTask.vue` — отсутствует :key в v-for
- **Строка:** ~11
- **Проблема:** `<ItemTask :task v-for="task in getTasks(section.uuid)" />` — нет `:key`
- **Последствие:** Vue не может корректно отслеживать элементы при переупорядочивании, баги при drag-and-drop и анимациях
- **Направление:** Добавить `:key="task.uuid"`
- [ ] Исправлено

---

## HIGH — нужно исправить в ближайшее время

### 3. `ItemTask.vue` — unsafe JSON.parse
- **Строка:** ~18
- **Проблема:** `JSON.parse(event.dataTransfer.getData("application/json"))` без `as ITask`
- **Последствие:** Потеря типобезопасности, IDE не подскажет поля, возможны runtime ошибки
- **Направление:** Добавить type assertion `as ITask`
- [ ] Исправлено

### 4. Drag-and-drop логика размазана по компонентам
- **Файлы:** `ItemSection.vue`, `ItemTask.vue`
- **Проблема:** Drag-логика дублируется и не переиспользуется. Нарушение DRY и SRP
- **Последствие:** Сложно поддерживать, легко допустить несогласованность
- **Направление:** Вынести в composable `useDragAndDrop` (см. TODO.md)
- [ ] Исправлено

## MEDIUM — улучшения качества кода

### 5. `InlineEdit.vue` — лишний crypto.randomUUID()
- **Строка:** ~35
- **Проблема:** `const name = crypto.randomUUID()` генерируется при каждом маунте для атрибута `name` инпута
- **Последствие:** Избыточная генерация UUID, нет реальной необходимости
- **Направление:** Использовать статическую строку или `useId()` из Vue 3.5+
- [ ] Исправлено

### 6. `task.ts` (Pinia store) — генераторы не сбрасываются
- **Строки:** ~28-36
- **Проблема:** `addNameIncrement` и `addDescIncrement` — генераторы, которые бесконечно инкрементируют числа и не сбрасываются между сессиями
- **Последствие:** После долгого использования задачи будут называться "Задача 9999..."
- **Направление:** Убрать генераторы, использовать timestamp или считать текущее количество задач
- [ ] Исправлено

### 7. `vite-env.d.ts` — any в типе компонента
- **Строка:** ~5
- **Проблема:** Использование `any` в декларации Vue-компонента
- **Направление:** Заменить на типизированный `DefineComponent`
- [ ] Исправлено

### 8. `HomeView.vue` — бизнес-логика в template
- **Проблема:** Кнопка добавления секции содержит inline handler с вызовом store
- **Направление:** Вынести в отдельную функцию в `<script setup>`
- [ ] Исправлено

## LOW — рекомендации

### 9. Нет error boundaries
- **Проблема:** Нет обработки ошибок на уровне компонентов. Если drag-and-drop выбросит ошибку — приложение упадёт
- **Направление:** Добавить `onErrorCaptured` или Vue error handler
- [ ] Исправлено

### 10. Нет переиспользуемых типов для drag events
- **Проблема:** `DragEvent` обрабатывается без кастомных типов для payload
- **Направление:** Создать интерфейс `DragPayload<T>` для типизации `dataTransfer`
- [ ] Исправлено

### 11. Отсутствует composable для работы со store
- **Проблема:** Компоненты напрямую импортируют и используют store. При переходе на Tauri backend придётся менять каждый компонент
- **Направление:** Создать service layer / composable-обёртки между компонентами и store
- [ ] Исправлено
