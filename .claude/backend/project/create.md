# create_project

Создание нового проекта. Автоматически создаёт дефолтные статусы и приоритеты.

## Tauri-команда

```
create_project
```

## Входные данные

Объект `CreateProject`:

```json
{
  "name": "Мой проект",
  "description": "Описание проекта"
}
```

| Поле          | Тип      | Обязательное | По умолчанию | Описание           |
|---------------|----------|--------------|--------------|--------------------|
| `name`        | `string` | Да           | —            | Название проекта   |
| `description` | `string` | Нет          | `""`         | Описание проекта   |

## Выходные данные

Объект `Project` — созданный проект (см. [get_all.md](./get_all.md)).

## Побочные эффекты

При создании проекта автоматически создаются:

### Дефолтные статусы (канбан-колонки)

| name       | color     | position |
|------------|-----------|----------|
| К работе   | `#6366f1` | 0        |
| В работе   | `#f59e0b` | 1        |
| Сделано    | `#22c55e` | 2        |

### Дефолтные приоритеты

| name     | color     | level |
|----------|-----------|-------|
| Низкий   | `#22c55e` | 0     |
| Средний  | `#eab308` | 1     |
| Высокий  | `#f97316` | 2     |
| Срочный  | `#ef4444` | 3     |

## Логика

1. Генерируется UUID v4
2. `position` = MAX(position) + 1 среди всех проектов
3. `created_at` и `updated_at` = текущее UTC-время
4. INSERT проекта
5. INSERT 3 статусов
6. INSERT 4 приоритетов
7. Возврат созданного проекта

## Ошибки

| Ошибка              | Когда                    |
|---------------------|--------------------------|
| `Ошибка базы данных` | Проблемы с SQLite-запросом |

## Пример вызова (TypeScript)

```ts
const project = await invoke<Project>("create_project", {
  data: { name: "Новый проект" }
});
```
