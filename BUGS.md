# Баги и проблемы кода — Chrono

## Баги

### 1. `editSection` не работает для первой секции
- **Файл:** `src/stores/section.ts:34`
- **Проблема:** `findIndex()` возвращает `0` для первого элемента, а `if (section)` считает `0` ложным значением. Первую секцию невозможно отредактировать.
- **Исправление:** Заменить `if (section)` на `if (section !== -1)`

### 2. Общий `changeSectionName` на все секции
- **Файл:** `src/App.vue:22`
- **Проблема:** Один `ref(false)` управляет режимом редактирования для всех секций в `v-for`. Клик на любую секцию открывает редактирование у всех одновременно.
- **Исправление:** Использовать `ref<string | null>(null)` с UUID активной секции, или `Map<string, boolean>`.

### 3. Общий `modelName` на все секции
- **Файл:** `src/App.vue:27`
- **Проблема:** Один `v-model` для всех инпутов в `v-for`. Ввод в одной секции перезаписывает значение для всех.
- **Исправление:** Привязать к конкретной секции (объект `Record<string, string>` по UUID).

### 4. `{{ task }}` рендерит объект
- **Файл:** `src/App.vue:56`
- **Проблема:** Выводится `[object Object]` вместо содержимого задачи.
- **Исправление:** Заменить на `{{ task.name }}` / `{{ task.description }}` или вынести в компонент `TaskCard`.

## Качество кода

### 5. `console.log` в продакшн-коде
- **Файл:** `src/stores/section.ts:37`
- **Исправление:** Удалить `console.log(sections.value[section]);`

### 6. `let` вместо `const`
- **Файлы:** `src/stores/task.ts:13`, `src/stores/section.ts:18`, `src/stores/section.ts:34`
- **Проблема:** Переменные не переназначаются, но объявлены через `let`.
- **Исправление:** Заменить на `const`.

### 7. Генераторы для ID задач — оверинжиниринг
- **Файл:** `src/App.vue:13-20`
- **Проблема:** Генераторы сбрасываются при ремаунте. В сторах уже используется `crypto.randomUUID()` — зачем второй механизм?
- **Исправление:** Убрать генераторы, использовать UUID или простой счётчик в сторе.

## Производительность

### 8. `getTasks()` без мемоизации
- **Файл:** `src/stores/task.ts:8`
- **Проблема:** `filter()` вызывается при каждом ре-рендере, создавая новый массив. Для каждой секции — отдельный вызов.
- **Исправление:** Использовать `computed` или группировать задачи в `Map<sectionUuid, ITask[]>`.

## Архитектура

### 9. Весь UI в одном `App.vue`
- **Файл:** `src/App.vue`
- **Проблема:** Нет декомпозиции. Вся логика и шаблон в корневом компоненте.
- **Исправление:** Вынести в компоненты: `SectionCard`, `TaskCard`, `AddSectionButton`.

### 10. Нет валидации ввода
- **Файлы:** `src/stores/section.ts`, `src/stores/task.ts`, `src/App.vue`
- **Проблема:** Можно создать секцию/задачу с пустым именем. `addSection('some')` — захардкожено.
- **Исправление:** Добавить проверку на пустую строку перед добавлением.

## Безопасность

### 11. CSP отключён
- **Файл:** `src-tauri/tauri.conf.json:21`
- **Проблема:** `"csp": null` — Content Security Policy полностью отключён.
