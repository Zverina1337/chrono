## КРИТИЧЕСКИЕ БАГИ

### BUG-01 — `Button.vue:9` — Неправильный `v-bind` (атрибуты не передаются)
**Файл:** `src/components/Button.vue`
**Проблема:** `v-bind:attrs="attrs"` создаёт HTML-атрибут с именем `attrs`, а не разворачивает объект атрибутов. Все пробрасываемые атрибуты (классы, обработчики) молча игнорируются.
**Принцип:** OCP — компонент должен быть расширяемым через атрибуты без изменения кода.
**Исправление:**
```vue
<!-- ❌ Неверно -->
<button v-bind:attrs="attrs">

<!-- ✅ Верно -->
<button v-bind="attrs">
```

---

### BUG-02 — `Button.vue` — `inheritAttrs` не отключён при ручной передаче атрибутов
**Файл:** `src/components/Button.vue`
**Проблема:** `useAttrs()` используется для ручной передачи атрибутов, но `inheritAttrs: false` не задан. Когда BUG-01 будет исправлен, атрибуты применятся дважды — автоматически Vue и вручную через `v-bind`.
**Исправление:** Добавить `defineOptions({ inheritAttrs: false })` в `<script setup>`.

---

### BUG-03 — `InlineEdit.vue` — Нет возможности выйти из режима редактирования
**Файл:** `src/components/InlineEdit.vue`
**Проблема:** `handleEdit` закомментирован (строки 26–29). После клика компонент переходит в режим редактирования, но закрыть его нельзя — нет обработчика `blur`, `Enter` или `Escape`.
**Принцип:** SRP — компонент должен управлять полным циклом своего состояния.
**Исправление:** Раскомментировать / доработать `handleEdit` и навесить обработчики:
```vue
<input
  v-model="editableValue"
  @blur="handleEdit"
  @keydown.enter="handleEdit"
  @keydown.escape="handleEdit"
/>
```

---

### BUG-04 — `ListSection.vue:53` — `handleAddSection` вызывается без аргументов
**Файл:** `src/components/ListSection.vue`
**Проблема:** `@click="handleAddSection"` — Vue передаёт `MouseEvent` как первый аргумент вместо строки `name`. Секция создаётся с именем `[object MouseEvent]`.
**Исправление:**
```vue
<!-- ❌ Неверно -->
<Button @click="handleAddSection">Добавить секцию</Button>

<!-- ✅ Временная заглушка -->
<Button @click="handleAddSection('Новая секция')">Добавить секцию</Button>
```

---

### BUG-05 — `HomeView.vue:17,34` — Захардкоженное имя секции `'some'`
**Файл:** `src/views/HomeView.vue`
**Проблема:** `addSection('some')` вызывается в двух местах с бессмысленной заглушкой. Пользователь не может задать имя при создании секции.
**Исправление:** Реализовать модальный диалог или inline-ввод для ввода имени новой секции.

## ПРЕДУПРЕЖДЕНИЯ

### WARN-01 — `ListSection.vue:35` — Отсутствует `:key` в `v-for`
**Файл:** `src/components/ListSection.vue`
**Проблема:** `v-for="section in sections"` без `:key`. Vue не может оптимально переиспользовать DOM-элементы, возможны баги при удалении/сортировке.
**Исправление:**
```vue
<div v-for="section in sections" :key="section.uuid" ...>
```

---

### WARN-02 — `ListSection.vue:47` — Второй `v-if` вместо `v-else`
**Файл:** `src/components/ListSection.vue`
**Проблема:** Два взаимоисключающих условия — `v-if="sections.length !== 0"` и `v-if="sections.length === 0"`. Оба условия вычисляются при каждом рендере, хотя второй блок — очевидный `v-else`.
**Исправление:**
```vue
<section v-if="sections.length !== 0" ...>
<section v-else ...>
```

---

### WARN-03 — `ItemSection.vue:3` — Неиспользуемые импорты
**Файл:** `src/components/ItemSection.vue`
**Проблема:** `shallowRef` и `ShallowRef` импортированы, но нигде не применяются. ESLint должен это поймать.
**Исправление:** Удалить из импорта.

---

### WARN-04 — `HomeView.vue:32–37` — Нативный `<button>` вместо компонента `Button`
**Файл:** `src/views/HomeView.vue`
**Проблема:** Дублирующая кнопка "Добавить секцию" использует нативный `<button>` со своими стилями вместо кастомного `Button`. Нарушение DRY, несогласованный UI.
**Принцип:** DRY — один компонент для кнопок.
**Исправление:** Заменить на `<Button>` или удалить дублирование.

## БЕЗОПАСНОСТЬ

### SEC-01 — `tauri.conf.json:21` — CSP отключён
**Файл:** `src-tauri/tauri.conf.json`
**Проблема:** `"csp": null` — Content Security Policy полностью отключён. Для десктопного приложения с локальными данными это допустимо на этапе разработки, но должно быть настроено перед релизом.
**Исправление:** Задать минимальный CSP при добавлении внешних ресурсов:
```json
"csp": "default-src 'self'; script-src 'self'"
```
